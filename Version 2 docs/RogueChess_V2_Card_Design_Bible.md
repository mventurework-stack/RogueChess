# Rogue Chess V2 -- Card Design Bible

## Purpose

This document defines how cards are designed and balanced in Rogue
Chess.

Cards are intended to increase tactical depth without replacing the
importance of positioning, army composition, or resource management.

------------------------------------------------------------------------

# Core Philosophy

Cards should:

-   Reward planning.
-   Create interesting tactical decisions.
-   Encourage different playstyles.
-   Never decide a match by themselves.

A player should win because they used a card well, not because they drew
a stronger card.

------------------------------------------------------------------------

# Core Principles

## 1. Cards Are Temporary

Cards never permanently modify units.

No:

-   Permanent health
-   Permanent damage
-   Permanent movement
-   Permanent abilities
-   Experience
-   Levels

Temporary effects only.

------------------------------------------------------------------------

## 2. Position First

A good position should always beat a poorly played powerful card.

Cards enhance positioning rather than replace it.

------------------------------------------------------------------------

## 3. One Card Per Turn

Players may normally play one card during their turn.

This keeps turns readable and prevents explosive combo chains.

------------------------------------------------------------------------

## 4. Scrap Is the Limiting Resource

Cards cost Scrap.

Choosing when to spend or save Scrap is a strategic decision.

Strong cards should naturally cost more.

------------------------------------------------------------------------

# Card Categories

## Movement

Examples:

-   Sprint (+1 movement)
-   Jet Boost (ignore blocking this turn)
-   Reposition (move an adjacent ally)

Purpose: Create tactical opportunities.

------------------------------------------------------------------------

## Combat

Examples:

-   Power Shot (+1 damage)
-   Rapid Fire (Shooter attacks twice)
-   Precision Strike (ignore cover or armor this attack)

Purpose: Improve attacks without making them automatic kills.

------------------------------------------------------------------------

## Defense

Examples:

-   Shield Matrix
-   Brace
-   Emergency Armor

Purpose: Help a unit survive one key moment.

------------------------------------------------------------------------

## Repair

Examples:

-   Repair Kit
-   Emergency Patch
-   Nanobots

Purpose: Restore health, never above maximum.

------------------------------------------------------------------------

## Control

Examples:

-   Signal Jam
-   EMP Pulse
-   Sensor Disruption

Purpose: Manipulate enemy options without removing player agency.

------------------------------------------------------------------------

## Deployment

Examples:

-   Build Buddy
-   Deploy Drone
-   Emergency Reinforcements

Purpose: Temporarily increase board presence while respecting game
balance.

------------------------------------------------------------------------

## Economy

Examples:

-   Salvage
-   Emergency Scrap
-   Resource Cache

Purpose: Generate or manipulate Scrap.

------------------------------------------------------------------------

# Card Power Budget

Every card should excel in one area only.

If a card:

-   Deals damage,
-   Moves a unit,
-   Repairs,
-   Generates Scrap,
-   Summons,

it generally should not do several of these at once.

Simple cards are easier to understand and balance.

------------------------------------------------------------------------

# Unit Synergy

Cards should support units rather than replace them.

Examples:

Buddy: - Sprint - Evasive Maneuvers

Shooter: - Rapid Fire - Target Lock

Tank: - Brace - Shield Matrix

Hacker: - Signal Boost - EMP Pulse

Commander: - Rally - Tactical Orders

------------------------------------------------------------------------

# Avoid These Designs

Do not create cards that:

-   Permanently improve units.
-   Eliminate meaningful counterplay.
-   Remove multiple enemy turns.
-   Instantly destroy powerful units without setup.
-   Win the game immediately.
-   Require long rule explanations.

------------------------------------------------------------------------

# Resource Philosophy

A player should constantly choose between:

-   Playing a card now.
-   Saving Scrap for a stronger card later.

There should rarely be an obvious correct answer.

------------------------------------------------------------------------

# Expansion Rules

When adding a new card, ask:

1.  What tactical decision does it create?
2.  Does it reinforce positioning?
3.  Is the effect temporary?
4.  Does it support an existing strategy?
5.  Is there reasonable counterplay?

If any answer is "no", redesign the card.

------------------------------------------------------------------------

# Success Criteria

A healthy card system means:

-   Players can win without drawing a specific card.
-   Cards create memorable tactical moments.
-   Army composition remains more important than card luck.
-   Scrap management is a meaningful strategic choice.
-   Card effects stay simple, readable, and temporary.

The ideal Rogue Chess card is easy to understand, useful in many
situations, and powerful only when played at the right moment.
